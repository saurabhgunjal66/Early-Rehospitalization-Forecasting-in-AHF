# AHF Rehospitalization Risk Predictor

## Overview

This is a Clinical Decision Support System designed to predict 30-day acute heart failure (AHF) readmission risk. The application uses machine learning models to analyze clinical parameters including NT-proBNP levels, body weight, and ultrasound parameters to assess patient rehospitalization risk. The system provides healthcare professionals with risk scores and visualizations to support clinical decision-making for AHF patients.

## User Preferences

Preferred communication style: Simple, everyday language.

## System Architecture

### Application Framework
- **Frontend**: Streamlit web application framework for interactive clinical interface
- **Rationale**: Streamlit chosen for rapid development of data-driven applications with minimal frontend code, ideal for medical decision support tools that require real-time interaction with ML models
- **Pros**: Fast prototyping, built-in widgets for data input, integrated visualization support
- **Cons**: Limited customization compared to full web frameworks, single-user session state management

### Machine Learning Architecture
- **Model Stack**: Ensemble approach combining Logistic Regression and XGBoost models
- **Rationale**: Dual-model ensemble provides interpretability (Logistic Regression) alongside high performance (XGBoost), critical for clinical applications requiring both accuracy and explainability
- **Feature Engineering**: 14 clinical features including demographics, biomarkers (NT-proBNP, creatinine), ultrasound parameters (B-line score, IVC collapsibility), and comorbidities
- **Class Balancing**: SMOTE (Synthetic Minority Over-sampling Technique) used to handle imbalanced readmission data
- **Pros**: Balanced interpretability and performance, handles class imbalance
- **Cons**: Requires periodic retraining with real clinical data

### Data Management
- **Storage**: SQLite database for local persistence
- **Schema Design**: 
  - `assessments` table: Stores patient evaluations with clinical parameters and risk predictions
  - `model_performance` table: Tracks model metrics over time
- **Rationale**: SQLite chosen for simplicity and zero-configuration deployment, suitable for single-instance clinical workstations
- **Alternatives Considered**: PostgreSQL for multi-user scenarios (may be added later)
- **Pros**: No server setup, file-based portability, ACID compliance
- **Cons**: Not suitable for concurrent multi-user access

### Data Generation & Training
- **Synthetic Data**: Custom generator creates realistic AHF patient records based on clinical literature distributions
- **Purpose**: Enables model training and testing without requiring actual patient data during development
- **Parameters**: Age, weight, biomarkers, ultrasound measurements, comorbidities generated with clinically realistic distributions
- **Rationale**: Addresses data privacy concerns and provides immediate system functionality for demonstration
- **Note**: Production deployment requires retraining with actual de-identified patient data

### Visualization Layer
- **Plotting Libraries**: Plotly for interactive charts, Matplotlib/Seaborn for statistical visualizations
- **Risk Display**: Gauge charts with color-coded risk levels (Low/Moderate/High)
- **Rationale**: Interactive visualizations improve clinical decision-making by providing intuitive risk communication
- **Components**: Risk gauges, ROC curves, feature importance plots, trend analysis

### Model Persistence
- **Serialization**: Pickle-based model saving/loading
- **Caching**: Streamlit's `@st.cache_resource` decorator for component initialization
- **Rationale**: Prevents redundant model retraining across user sessions, improves application responsiveness

## External Dependencies

### Core ML & Data Science Libraries
- **scikit-learn**: Model training, preprocessing (StandardScaler), evaluation metrics
- **XGBoost**: Gradient boosting classifier for high-performance predictions
- **imbalanced-learn**: SMOTE implementation for handling class imbalance
- **pandas/numpy**: Data manipulation and numerical operations
- **scipy**: Statistical distributions for synthetic data generation

### Visualization & UI
- **Streamlit**: Web application framework and UI components
- **Plotly**: Interactive plotting library for risk gauges and charts
- **Matplotlib/Seaborn**: Statistical visualization support

### Data Storage
- **SQLite3**: Built-in Python database library for local data persistence
- **Database File**: `ahf_predictions.db` (created automatically on first run)

### Development Tools
- **pickle**: Model serialization (Python standard library)
- **warnings**: Suppression of non-critical alerts during model training

### Future Integration Considerations
- System designed to accommodate PostgreSQL migration for multi-user deployments
- API endpoints could be added for EHR (Electronic Health Record) integration
- FHIR (Fast Healthcare Interoperability Resources) compatibility possible for healthcare system integration
