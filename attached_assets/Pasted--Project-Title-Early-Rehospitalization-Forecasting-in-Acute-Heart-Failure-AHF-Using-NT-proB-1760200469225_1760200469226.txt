 **Project Title:** **Early Rehospitalization Forecasting in Acute Heart Failure (AHF) Using NT-proBNP, Body Weight, and Ultrasound via AI** --- ## 🩺 **1. Introduction** Acute Heart Failure (AHF) is a serious cardiovascular condition that frequently leads to emergency hospitalization. Even after appropriate in-hospital treatment, approximately **25% of AHF patients are readmitted within 30 days** of discharge. These early readmissions not only increase healthcare costs but also indicate incomplete recovery or residual fluid overload. Traditional discharge assessment tools such as **NT-proBNP levels** and **body weight** are useful but have limited predictive value for short-term readmission. In contrast, **point-of-care ultrasound (POCUS)** provides additional diagnostic information through metrics such as: * **Lung B-line Score:** Indicates pulmonary congestion (fluid in lungs). * **Inferior Vena Cava (IVC) Collapsibility Index:** Reflects central venous pressure and fluid status. By integrating these clinical and ultrasound-based parameters with **Artificial Intelligence (AI)**, this project aims to create a **predictive decision-support system** to identify patients at high risk of early rehospitalization after AHF discharge. --- ## 🎯 **2. Project Objectives** 1. **Predictive Model Development:** Build an AI-based system to predict the likelihood of 30-day rehospitalization in AHF patients. 2. **Data Integration:** Combine traditional clinical parameters (NT-proBNP, body weight) with ultrasound-derived features (B-line score, IVC index). 3. **Model Evaluation:** Train and compare multiple ML algorithms — Logistic Regression, Random Forest, and XGBoost. 4. **User Interface:** Create a simple and intuitive web interface using **Streamlit**. 5. **Data Storage:** Store patient data locally and securely using **SQLite database** for offline accessibility. 6. **Clinical Impact:** Support clinicians in making more accurate and data-driven discharge decisions. --- ## ⚙️ **3. Machine Learning Methodology** ### Step 1: **Data Preprocessing** * Handle missing values (if any). * Normalize features using **StandardScaler** to improve model performance. * Split dataset into **80% training** and **20% testing** sets. ### Step 2: **Model Selection** We compare two supervised ML algorithms: 1. **Logistic Regression** – interpretable, well-suited for binary classification. 2. **Random Forest / XGBoost** – ensemble methods providing higher accuracy and handling non-linearities. ### Step 3: **Model Training** Models are trained to classify patient rehospitalization risk (0 = low, 1 = high). Key training parameters include: * Logistic Regression: default regularization (L2 penalty). * Random Forest: 100 decision trees, random_state=42 for reproducibility. ### Step 4: **Model Evaluation** Metrics used: * **Accuracy** * **Precision** * **Recall** * **F1-Score** * **Confusion Matrix** Example output: ``` 🔹 Logistic Regression Accuracy: 82% 🔹 Random Forest Accuracy: 86% ``` ### Step 5: **Model Persistence** * Save trained models using `joblib`. * Store both the model and the feature scaler for use in the Streamlit app. --- ## 💻 **6.
